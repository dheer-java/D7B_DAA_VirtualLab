Algorithm at a glance

Dijkstra's Algorithm, proposed by Edsger W. Dijkstra in 1956, is a single-source shortest path algorithm. It works by iteratively picking the unvisited node with the smallest tentative distance, then relaxing (updating) the distances of all its neighbors. This greedy strategy guarantees correctness on graphs with non-negative weights.

Why negative weights break Dijkstra

Important: Dijkstra's algorithm assumes that once a node is marked visited, its shortest distance is final. A negative-weight edge can invalidate this assumption — a later path through a negative edge might yield a shorter distance to an already-finalized node. Use Bellman-Ford for graphs with negative weights (but no negative cycles).

Pseudocode

DIJKSTRA(G, w, s):
  for each vertex v in G.V:
    v.d = ∞
    v.parent = NIL
  s.d = 0
  Q = min-priority queue of all vertices keyed by .d
  while Q ≠ ∅:
    u = EXTRACT-MIN(Q)
    for each vertex v in G.Adj[u]:
      if v.d > u.d + w(u, v):
        v.d = u.d + w(u, v)
        v.parent = u
        DECREASE-KEY(Q, v, v.d)

Applications

  1. Digital mapping (GPS) — finding the fastest route between two geographic locations
  2. IP routing protocols (OSPF) — routers use shortest-path trees to forward packets efficiently
  3. Robotic path planning — autonomous robots navigate obstacle maps using weighted graphs
  4. Telecommunication networks — minimizing latency or bandwidth cost across network topologies
  5. Social networks — computing degrees of separation and influence propagation paths
